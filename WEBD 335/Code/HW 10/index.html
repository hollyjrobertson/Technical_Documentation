[2 point] Describe how AngularJS uses ViewModel and why this is different than the way JQuery works?

  • The ViewModel in the Model-View-ViewModel is the glue between the model and view (back-end/front-end). It is responsible for maintaining the relationship between teh view and model. But the main difference between this structure and the JQuery structure is that instead of having the jQuery controller attaching event listeners onto front-end actions, the ViewModel requires the front-end/view to alert the model of an action via the ViewModel. Question 4

[2 point] What are the two common approaches to structuring an AngularJS application? Describe them.

  • Sort By Type
    • This one gets more difficult as the application gets bigger. When you want to find a specific view and its controller, they are in different folders. This can be a good one to start with since it is quite easy to shift to a more advanced structure. Source
  • Sort By Feature
    • All of the layout views and controllers go in the layout folder, the admin content goes in the admin folder, and the services that are used by all of the areas go in the services folder. The idea is that when you are looking for the code that makes a feature work, it is located in one place. Services are a bit different as they "service" many features. Source
